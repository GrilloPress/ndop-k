Design hypotheses

Viewing preferences

Previous "Type-2" opt outs will be migrated over and those users may use this service to view their opt-out.

Users who have opted-out may wish to return to view the status of their preference.

If we provide a previous opt-out user with a notification of their current status after they have verified themselves
Then users will know whether they have previously set a preference
Because that information will be provided in the context after logging into the system

If we provide a previous opt-out user a page that shows their preference before the question
Then users will know whether they have previously set a preference
Because that information will be provided in that context and allow them to edit if they wish

If we provide a previous opt-out user a page that shows their preference on the question page
Then users will know whether they have previously set a preference
Because that information will be provided in that context and allow them to update their preference if they wish
